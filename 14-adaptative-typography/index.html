01 — clamp()

Three values.
One smooth range.

The clamp(min, preferred, max) function constrains a value between a minimum and maximum, with a preferred middle value that scales fluidly.

For typography, this means you set a smallest acceptable size, a largest comfortable size, and a fluid preferred value tied to the viewport. The browser handles everything in between.

min 1rem
preferred 2.5vw
max 3rem
font-size: clamp(1rem, 2.5vw, 3rem);

02 — calc()

Math in your stylesheet.

calc() lets you mix units in a mathematical expression. For fluid typography, it's used to define a linear scale between two sizes across a viewport range.

The formula calculates how much the font should grow per pixel of viewport width, creating a straight line interpolation between your minimum and maximum sizes.

Fluid font scale formula

font-size: calc( 16px + (24 - 16) * (100vw - 320px) / (1280px - 320px) );
  • 16px — min font size
  • 24 — max font size (unitless)
  • 320px — min viewport
  • 1280px — max viewport

Paragraph text

calc(14px + (18 - 14) * …) 14px → 18px

Section title

calc(20px + (36 - 20) * …) 20px → 36px

Big display

calc(32px + (72 - 32) * …) 32px → 72px
03 — Viewport Units

vw, vh, vmin, vmax

Viewport units let type scale directly with the browser window. While powerful, pure viewport units have no minimum or maximum — a heading sized at 5vw becomes unreadably small on mobile. That's why they're best combined with clamp() or used sparingly for decorative type.

Ag
vw

1% of viewport width. Grows horizontally.

font-size: 5vw
Ag
vh

1% of viewport height. Rare for body text.

font-size: 5vh
Ag
vmin

1% of the smaller viewport dimension.

font-size: 5vmin
Ag
vmax

1% of the larger viewport dimension.

font-size: 5vmax
⚠
Accessibility note: Pure viewport units don't respect user browser font size preferences. Always combine them with relative units or use clamp() with rem-based min/max values.

How fluid typography changed the web

By the CSS Grid era · 5 min read

For years, responsive typography meant writing separate font sizes for each breakpoint. Today, a single declaration can handle every screen size gracefully.

The introduction of clamp() in modern browsers marked a turning point. Instead of defining type at 16px on mobile, then overriding it at 18px on tablet, then 20px on desktop — you define a range, and the browser interpolates smoothly.

Combined with calc() for precise linear interpolation and viewport units for display headings, designers now have the tools to build type systems that feel intentional at any size — from a 320px phone to a 2560px monitor.

"Good typography is invisible. Fluid typography is effortless."
